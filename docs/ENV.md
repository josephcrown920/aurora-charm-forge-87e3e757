# Environment & Secrets

Aurora runs on Lovable Cloud. There are **two** kinds of configuration:

1. **Build-time env vars** (`.env`, auto-managed by Lovable Cloud) — exposed to the client via `import.meta.env.VITE_*`.
2. **Runtime secrets** (Project Settings → Secrets) — exposed to server functions via `process.env.*`. Never bundled to the client.

> **Do not edit `.env` manually.** It is regenerated by the Cloud integration.

---

## Build-time (`.env`)

| Variable | Purpose |
|---|---|
| `VITE_SUPABASE_URL` | Supabase project URL (public) |
| `VITE_SUPABASE_PUBLISHABLE_KEY` | Anon/publishable key (public, RLS enforced) |
| `VITE_SUPABASE_PROJECT_ID` | Project ref for client tooling |
| `SUPABASE_URL` | Mirror for server runtime |
| `SUPABASE_PUBLISHABLE_KEY` | Mirror for server runtime |

Safe to commit — these are publishable values, protected by Row-Level Security.

---

## Runtime secrets

Managed in **Project Settings → Secrets** (or via the agent's `add_secret` tool). Read inside `createServerFn().handler()` — never at module top-level.

### Supabase (managed)

| Secret | Used by |
|---|---|
| `SUPABASE_SERVICE_ROLE_KEY` | `client.server.ts` admin client (bypasses RLS) |
| `SUPABASE_DB_URL` | Direct Postgres access for migrations |

### AI providers — fallback chain

Order in `src/lib/llm-fallback.server.ts`: **Lovable AI → Gemini → OpenAI → OpenRouter**.

| Secret | Provider | Notes |
|---|---|---|
| `LOVABLE_API_KEY` | Lovable AI Gateway | Primary; managed — rotate via `lovable_api_key--rotate_lovable_api_key` |
| `GEMINI_API_KEY` | Google Gemini direct | Fallback #1 |
| `OPENAI_API_KEY` | OpenAI direct | Fallback #2 |
| `OPENROUTER_API_KEY` | OpenRouter | Fallback #3 |
| `INFERENCE_API_KEY` | Inference.ai GPU workers | Used by orchestrator workers |

### Media / video

| Secret | Provider |
|---|---|
| `FAL_KEY` | fal.ai (Seedance, Seedream, some Kling endpoints) |
| `KLING_ACCESS_KEY` | Kling AI direct (JWT access id) |
| `KLING_SECRET_KEY` | Kling AI direct (JWT secret) |
| `SYNC_API_KEY` | Sync.so lip-sync |

### Payments & infra

| Secret | Purpose |
|---|---|
| `PAYSTACK_SECRET_KEY` | Paystack server API + webhook signature verify |
| `CRON_SECRET` | Shared secret for `/api/public/*` cron endpoints |

## Adding a new secret

1. Use the `add_secret` tool (agent) or Project Settings → Secrets (UI).
2. Read it inside a server function:
   ```ts
   export const fn = createServerFn({ method: "POST" }).handler(async () => {
     const key = process.env.MY_NEW_KEY!;
     // ...
   });
   ```
3. Never reference `process.env.*` in client code or at module scope of shared files — it will be `undefined` in the browser.

## Rotating secrets

- Standard secrets: `update_secret` tool or UI.
- `LOVABLE_API_KEY`: **must** use the rotate tool (`lovable_api_key--rotate_lovable_api_key`); it is managed.
